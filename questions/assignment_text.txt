Project Background:  
 
You must  design and implement a web application based on the following  specification :  
 
• Student Routine Organizer  is a web -based application that helps users, particularly 
students, manage and improve their daily routines in a centralized system. The 
application is developed using PHP and MySQL with a three -tier architecture and 
includes secure user registration, log in, and full CRUD functions for each module. All 
data is linked to individual users through a shared database, enabling users to track and 
manage different aspects of their daily activities in an organised manner.  
The web application should cover the following FOUR (4)  modules:  
 
1. Exercise Tracker Module  
▪ Focus: Health and fitness activities.  
▪ This module allows users to record and manage their exercise activities, 
including workouts such as jogging, cycling, gym sessions, or 
swimming. Each record represents a workout session with details such 
as activity type, duration, calories burned, and dat e. 
▪ Basic Features : Add, view, edit, and delete exercise  records .

 2. Diary Journal Module  
▪ Focus: Personal reflections and mood tracking.  
▪ This module allows users to maintain a digital journal by recording their 
daily thoughts, experiences, and moods. Each record represents a journal 
entry with details such as title, content, mood status, and date.  
▪ Basic Features: Add, view, edit, and delete journal entries . 
 
3. Money Tracker Module  
▪ Focus: Personal finance management.  
▪ This module allows users to monitor their income and expenses by 
recording financial transactions. Each record represents an income or 
expense entry with details such as amount, category, description, 
transaction type, and date.  
▪ Basic Features: Add, view, edit, and delete financial records.  
 
4. Habit Tracker Module  
▪ Focus: Personal habit development.  
▪ This module allows users to create and monitor habits that support 
positive daily routines. Users can track habit completion over time and 
monitor their progress. Each record represents a habit with details such 
as habit name, target frequency, completion status, and date.  
▪ Basic Features: Add, view, edit, and delete habit records.  
 
• Functional Requirements : Students must design and develop the system using PHP 
and MySQL with database connectivity and full CRUD functions for all four modules. 
The application must include session -based access control, cookie usage, error 
handling, and a relational database design that links module records to the authenticated 
user. A main menu or dashboard must be provided for consistent access to all modules.  
Centralized User Management:  
▪ The system must support both Student and Admin user roles.  
▪ User authentication must be handled centrally and shared across all modules.  
▪ Student users can manage only their own records.  
▪ Admin users can view all registered users and basic system summaries.  
 
• Optional Feature Suggestion:  
▪ Additional navigation features, such as sorting, filtering, or searching, to 
improve record browsing.  
▪ Additional features based on concepts covered during the  practical lab  classes.  
▪ Additional innovative features that  demonstrate creativity, system complexity, 
or integration of concepts learned throughout the course.
