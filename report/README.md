# 📄 Report Writing Guidelines

## Format
- **Referencing:** IEEE Referencing System
- **Max Pages:** 30 (excluding cover page and references)
- **File Format:** PDF

## Required Sections (in order)

### 1. Title Page (Leader)
- Unit code: UCCD3243
- Course title: Server-Side Web Applications Development
- Course name (e.g., IA/IB/etc.)
- All group members' names and student IDs
- Practical group number
- Assignment title: Student Routine Organizer (RoutineFlow)
- Include marking scheme table on front page

### 2. Site Hierarchy & Navigation (Leader) — 2-3 pages
- Site map diagram showing all pages and their relationships
- Explanation of user navigation flow
- Cover both Student and Admin user flows

### 3. System Flowcharts (Each Member) — 2-3 pages each
- Each member creates flowchart for their module
- Must use proper flowchart notations (start/end ovals, process rectangles, decision diamonds)
- Include explanation of each step

### 4. Overview of Database Structure (Leader) — 2-3 pages
- ER Diagram showing all tables and relationships
- Explanation of foreign keys and data flow
- How user-specific data is stored and retrieved

### 5. Functional Requirements (Each Member) — 2-3 pages each
- List all features of your module
- Explain how each feature supports the module's purpose
- Include screenshots of key pages

### 6. References (All)
- IEEE format
- Each member adds references for their module
- Include textbook, PHP docs, and any API documentation used

## Tools Suggested
- **Diagrams:** Draw.io (free), Lucidchart, or Microsoft Visio
- **Report:** Google Docs (for collaboration) or Microsoft Word
- **Screenshots:** Use browser dev tools or Snipping Tool
